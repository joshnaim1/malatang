# Creator Strategy Playbook — v0 (baseline, hand-written)

This is the iteration-0 playbook injected verbatim into the Creator's system
prompt (SOW section 7, Level 1). It is intentionally general. The reflection
step (`creator/reflection.py`, Fireworks) rewrites it into `v1`, `v2`, ... using
strategies mined from the Creator's own verified wins — that rewrite is the
Level 1 self-improvement signal, not this file.

## Global rules

- Change the fewest lines that fix the root cause. Never refactor unrelated code.
- Output a unified diff that `git apply -p1` accepts: keep `a/` and `b/` path
  prefixes, correct `@@` hunk headers, and unchanged context lines.
- Fix only the file you were given. Do not invent new files or imports.
- One defect per attempt. If unsure between two edits, pick the smaller one.

## Per-bug-class tactics (seed heuristics)

- **syntax**: scan for the unbalanced token — missing `>` on a JSX close, an
  unclosed brace/paren, or an unterminated template literal. The fix is almost
  always a single character or single line.
- **off-by-one**: check loop bounds and slice indices. `<=` vs `<`, `i+1`,
  `length` vs `length-1`, and reduce initial values are the usual culprits.
- **null-handling**: add the missing guard — optional chaining (`?.`), a default
  (`?? fallback`), or an early return for empty/nullish input.
- **wrong-api**: verify the exact API contract (e.g. `Intl.NumberFormat`
  expects major units, `formatToParts` vs `format`, correct part `type`).
- **async**: add the missing `await`, ensure the enclosing function is `async`,
  and do not swallow rejected promises.

## Output discipline

Respond with strict JSON only: `root_cause`, `plan`, `reasoning` (one line),
`diff`. No prose, no markdown fences around the object.
